# ccBitTorrent - 高性能BitTorrentクライアント

Python asyncioで構築された、高度なピース選択アルゴリズム、並列メタデータ交換、最適化されたディスクI/Oを特徴とする、モダンで高性能なBitTorrentクライアント。

## 機能

### パフォーマンス最適化
- **非同期I/O**: 優れた並行性のための完全なasyncio実装
- **Rarest-First選択**: 最適なスワームヘルスためのインテリジェントなピース選択
- **エンドゲームモード**: より高速な完了のための重複リクエスト
- **リクエストパイプライン**: 深いリクエストキュー（ピアあたり16-64の未処理リクエスト）
- **Tit-for-Tat Choking**: 楽観的アンチョークによる公平な帯域幅割り当て
- **並列メタデータ**: 複数のピアからの同時ut_metadata取得
- **ディスクI/O最適化**: ファイル事前割り当て、バッチ書き込み、リングバッファステージング、メモリマップI/O
- **ハッシュ検証プール**: ワーカースレッド間での並列SHA-1検証

### 高度な設定
- **TOML設定**: ホットリロード付きの包括的な設定システム
- **トーレントごとの設定**: 個別のトーレント設定オーバーライド
- **レート制限**: グローバルおよびトーレントごとのアップロード/ダウンロード制限
- **戦略選択**: ラウンドロビン、rarest-first、または順次ピース選択
- **ストリーミングモード**: メディアファイルの優先度ベースのピース選択

### ネットワーク機能
- **UDPトラッカーサポート**: BEP 15準拠のUDPトラッカー通信
- **強化DHT**: 反復ルックアップ付きの完全なKademliaルーティングテーブル
- **ピア交換（PEX）**: BEP 11準拠のピア発見
- **接続管理**: 適応的ピア選択と接続制限
- **プロトコル最適化**: ゼロコピーパスによるメモリ効率的なメッセージ処理

## Xetプロトコル拡張（BEP XET）

Xetプロトコル拡張は、BitTorrentをコラボレーションに最適化された超高速で更新可能なピアツーピアファイルシステムに変換する重要な差別化要因です。BEP XETは以下を可能にします：

- **コンテンツ定義チャンキング**: 効率的な更新のためのGearhashベースのインテリジェントファイルセグメンテーション（8KB-128KBチャンク）
- **クロストーレント重複排除**: 複数のトーレント間でのチャンクレベルの重複排除
- **ピアツーピアCAS**: DHTとトラッカーを使用した分散コンテンツアドレッサブルストレージ
- **超高速更新**: 変更されたチャンクのみが再配布を必要とし、迅速な協調ファイル共有を可能にします
- **P2Pファイルシステム**: BitTorrentをコラボレーションに最適化された更新可能なピアツーピアファイルシステムに変換
- **マークルツリー検証**: 整合性のためのBLAKE3-256ハッシュとSHA-256フォールバック

[BEP XETの詳細 →](bep_xet.md)

### 可観測性
- **メトリクスエクスポート**: 監視のためのPrometheus互換メトリクス
- **構造化ロギング**: ピアごとのトレーシング付きの設定可能なロギング
- **パフォーマンス統計**: リアルタイムのスループット、レイテンシ、キュー深度の追跡
- **ヘルス監視**: 接続品質とピア信頼性スコアリング
- **ターミナルダッシュボード**: Textualベースのライブダッシュボード（Bitonic）
- **アラートマネージャー**: 永続化とCLI経由のテスト付きルールベースのアラート

## クイックスタート

### UVでのインストール

[astral.sh/uv](https://astral.sh/uv)からUVをインストールし、次にccBitTorrentをインストールします。

### メインエントリーポイント

**Bitonic** - メインターミナルダッシュボードインターフェース（推奨）:
- 起動: `uv run bitonic` または `uv run ccbt dashboard`

**btbt CLI** - 強化されたコマンドラインインターフェース:
- 起動: `uv run btbt`

**ccbt** - 基本的なCLIインターフェース:
- 起動: `uv run ccbt`
