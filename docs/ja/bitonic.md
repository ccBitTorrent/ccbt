# Bitonic - ターミナルダッシュボード

**Bitonic** は ccBitTorrent のメインエントリーポイントで、トレント、ピア、速度、システムメトリクスを監視および管理するためのライブで対話的なターミナルダッシュボードを提供します。

- エントリーポイント: [ccbt/interface/terminal_dashboard.py:main](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L3914)
- 定義場所: [pyproject.toml:81](https://github.com/yourusername/ccbittorrent/blob/main/pyproject.toml#L81)
- メインクラス: [ccbt/interface/terminal_dashboard.py:TerminalDashboard](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L3009)

## Bitonic の起動

専用のエントリーポイントを使用して Bitonic を起動:
```bash
uv run bitonic
```

または CLI 経由:
```bash
uv run ccbt dashboard
```

オプション付き:
```bash
# カスタム更新間隔（秒）
uv run bitonic --refresh 2.0

# アラートルール付き CLI 経由
uv run ccbt dashboard --rules /path/to/alert-rules.json
```

実装: [ccbt/cli/monitoring_commands.py:dashboard](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/cli/monitoring_commands.py#L20)

## 完全なユーザージャーニーの例

```
ユーザーアクションフロー:
─────────────────

1. ユーザーがメインダッシュボードで 'o'（高度な追加）を押す
   ↓
2. トレント追加画面のモーダルが表示
   ↓
3. ステップ 1: ユーザーがマグネットリンクを入力
   → 入力: "magnet:?xt=urn:btih:abc123..."
   → 検証: ✓ 有効なマグネット URI
   → 読み込み: トレントメタデータ
   → クリック: [次へ]
   ↓
4. ステップ 2: ユーザーが出力ディレクトリを入力
   → 入力: "/home/user/downloads"
   → クリック: [次へ]
   ↓
5. ステップ 3: ユーザーがファイルを選択
   → 選択: ファイル 0, 1, 3 (ファイルをスキップ 2)
   → 設定: ファイル 1 優先度を "高"
   → クリック: [次へ]
   ↓
6. ステップ 4: ユーザーがレート制限を設定
   → ダウンロード: 1000 KiB/s
   → アップロード: 500 KiB/s
   → クリック: [次へ]
   ↓
7. ステップ 5: ユーザーがキュー優先度を選択
   → 選択: "高"
   → クリック: [次へ]
   ↓
8. ステップ 6: ユーザーが再開を有効化
   → スイッチ: ON
   → クリック: [次へ]
   ↓
9. ステップ 7: ユーザーが重複排除付き Xet を有効化
   → Xet を有効化: ON
   → 重複排除: ON
   → P2P CAS: OFF
   → 圧縮: OFF
   → クリック: [次へ]
   ↓
10. ステップ 8: ユーザーが IPFS をスキップ
    → IPFS を有効化: OFF
    → クリック: [次へ]
    ↓
11. ステップ 9: ユーザーが自動スクレイプを有効化
    → 自動スクレイプ: ON
    → クリック: [次へ]
    ↓
12. ステップ 10: ユーザーが uTP を有効化
    → uTP を有効化: ON
    → クリック: [次へ]
    ↓
13. ステップ 11: ユーザーが NAT マッピングを有効化
    → NAT を有効化: ON
    → クリック: [送信]
    ↓
14. フォームが送信
    → オプション辞書が構築
    → モーダルが閉じられる
    → dashboard._process_add_torrent() 呼び出される
    → すべての指定されたオプションでトレントが追加
```

### キーバインド

#### メインダッシュボードのアクション
- `p` - トレントを一時停止
- `r` - トレントを再開
- `q` - 終了
- `i` - クイックトレント追加
- `o` - 高度なトレント追加
- `b` - トレントファイルを参照して追加
- `?` - ヘルプ

#### 設定
- `g` - グローバル設定
- `t` - トレント設定

#### 監視画面
- `s` - システムリソース
- `m` - パフォーマンスメトリクス
- `n` - ネットワーク品質
- `h` - 履歴トレンド
- `a` - アラートダッシュボード
- `e` - メトリクスエクスプローラー
- `x` - セキュリティスキャン

#### プロトコル管理
- `Ctrl+X` - Xet 管理
- `Ctrl+I` - IPFS 管理
- `Ctrl+S` - SSL 設定
- `Ctrl+P` - プロキシ設定
- `Ctrl+R` - スクレイプ結果
- `Ctrl+N` - NAT 管理
- `Ctrl+U` - uTP 設定

#### ナビゲーション
- `Ctrl+M` - ナビゲーションメニュー

## 監視画面

Bitonicは、メインダッシュボードからアクセス可能な専門的な監視画面を提供します。各画面は、詳細なメトリクスと可視化を備えた特定の監視ドメインに焦点を当てています。

### システムリソース画面

**アクセス**：メインダッシュボードから`s`キーを押す

**目的**：システムリソース使用率（CPU、メモリ、ディスク、ネットワークI/O）を表示します。

**機能**：
- プログレスバー付きのCPU使用率パーセンテージ
- プログレスバー付きのメモリ使用率パーセンテージ
- プログレスバー付きのディスク使用率パーセンテージ
- プロセス数
- ネットワークI/O統計（送信/受信バイト数）

**実装**：[ccbt/interface/terminal_dashboard.py:SystemResourcesScreen](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L690)

**ナビゲーション**：`Escape`または`q`キーを押してメインダッシュボードに戻る

### パフォーマンスメトリクス画面

**アクセス**：メインダッシュボードから`m`キーを押す

**目的**：MetricsCollectorとMetricsPluginからのパフォーマンスメトリクスを表示します。

**機能**：
- ピア接続数
- ダウンロード/アップロード速度
- ピース完了/失敗統計
- トラッカーリクエスト/レスポンス統計
- MetricsPluginからのイベント駆動メトリクス（利用可能な場合）
- メトリクス収集統計

**実装**：[ccbt/interface/terminal_dashboard.py:PerformanceMetricsScreen](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L811)

**データソース**：
- `MetricsCollector.get_performance_metrics()`
- `MetricsCollector.get_metrics_statistics()`
- `MetricsPlugin.get_aggregates()`（利用可能な場合）

**ナビゲーション**：`Escape`または`q`キーを押してメインダッシュボードに戻る

### ネットワーク品質画面

**アクセス**：メインダッシュボードから`n`キーを押す

**目的**：ピアと接続のネットワーク品質メトリクスを表示します。

**機能**：
- グローバルネットワーク統計
- トレントごとのネットワーク品質テーブル
- 視覚的インジケーター付きのピア接続品質メトリクス
- 接続品質スコア（0-100）

**実装**：[ccbt/interface/terminal_dashboard.py:NetworkQualityScreen](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L889)

**品質計算**：
- ピア速度とチョークステータスに基づく
- 視覚的インジケーター：████████（優秀）、██████░░（良好）、████░░░░（普通）、██░░░░░░（不良）

**ナビゲーション**：`Escape`または`q`キーを押してメインダッシュボードに戻る

### 履歴トレンド画面

**アクセス**：メインダッシュボードから`h`キーを押す

**目的**：スパークラインを使用してさまざまなメトリクスの履歴トレンドを表示します。

**機能**：
- 異なるメトリクス用の複数のスパークラインウィジェット

**追跡されるメトリクス**：

## 監視との統合

BitonicはccBitTorrentの監視システムと統合されています：

- [ccbt/plugins/base.py](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/plugins/base.py)によるプラグインシステム
- イベント駆動メトリクスのためのMetricsPlugin統合。参照：[ccbt/interface/terminal_dashboard.py:_get_metrics_plugin](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L626)
- システムメトリクストラッキング。参照：[ccbt/interface/terminal_dashboard.py:3001-3019](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L3001-L3019)

### メトリクスソース

Bitonicは複数のソースからのメトリクスを表示します：

1. **MetricsCollector**：システムレベルおよびパフォーマンスメトリクス
   - CPU、メモリ、ディスク使用率
   - ネットワークI/O統計
   - ピア接続、速度、ピース統計

2. **MetricsPlugin**：イベント駆動メトリクス
   - ピースダウンロード速度
   - トレント平均速度
   - イベントベースの集約

3. **AsyncSessionManager**：セッションレベルの統計
   - グローバルダウンロード/アップロードレート
   - トレントごとのステータス
   - ピア情報

### プラグインマネージャー統合

Bitonicはグローバルプラグインマネージャーシングルトンを使用してプラグインにアクセスします：
- `get_plugin_manager()`関数によるアクセス。参照：[ccbt/plugins/base.py:get_plugin_manager](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/plugins/base.py#L403)
- 複数の方法によるMetricsPlugin検出（PluginManager、イベントバス、セッション属性）
- プラグインが利用できない場合の適切な処理

## トラブルシューティング

### ダッシュボードが起動しない
1. Textualがインストールされているか確認：`uv pip install textual>=0.73.0`
2. ターミナルがUnicodeと色をサポートしているか確認
3. ターミナルのエラーメッセージを確認

実装はTextualの可用性を処理します：[ccbt/interface/terminal_dashboard.py:46-172](https://github.com/yourusername/ccbittorrent/blob/main/ccbt/interface/terminal_dashboard.py#L46-L172)

### パフォーマンスの問題
1. 更新間隔を増やす：`--refresh 2.0`または`R`を押して間隔を切り替え
2. コンパクトモードを使用：`c`を押して切り替え
3. 不要な場合は速度グラフを無効化

### データが表示されない
1. トレントがアクティブにダウンロードされていることを確認
2. ネットワーク接続を確認
3. ピア接続が確立されていることを確認
