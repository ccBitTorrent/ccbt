# ccBitTorrent - کلاینت BitTorrent با کارایی بالا

یک کلاینت BitTorrent مدرن و با کارایی بالا که با Python asyncio ساخته شده است، با الگوریتم‌های پیشرفته انتخاب قطعه، تبادل متادیتا موازی، و I/O دیسک بهینه‌شده.

## ویژگی‌ها

### بهینه‌سازی‌های عملکرد
- **I/O ناهمگام**: پیاده‌سازی کامل asyncio برای همزمانی برتر
- **انتخاب Rarest-First**: انتخاب هوشمند قطعه برای سلامت بهینه سوارم
- **حالت Endgame**: درخواست‌های تکراری برای تکمیل سریع‌تر
- **خط لوله درخواست**: صف‌های درخواست عمیق (16-64 درخواست معلق در هر همتا)
- **Tit-for-Tat Choking**: تخصیص عادلانه پهنای باند با unchoke خوش‌بینانه
- **متادیتا موازی**: واکشی همزمان ut_metadata از چندین همتا
- **بهینه‌سازی I/O دیسک**: پیش‌تخصیص فایل، نوشتن دسته‌ای، مرحله‌بندی ring-buffer، I/O نگاشت‌شده حافظه
- **استخر تأیید هش**: تأیید SHA-1 موازی در رشته‌های کارگر

### پیکربندی پیشرفته
- **پیکربندی TOML**: سیستم پیکربندی جامع با بارگذاری مجدد داغ
- **تنظیمات هر تورنت**: بازنویسی‌های پیکربندی تورنت فردی
- **محدودیت نرخ**: محدودیت‌های آپلود/دانلود جهانی و هر تورنت
- **انتخاب استراتژی**: انتخاب قطعه round-robin، rarest-first، یا ترتیبی
- **حالت استریمینگ**: انتخاب قطعه مبتنی بر اولویت برای فایل‌های چندرسانه‌ای

### ویژگی‌های شبکه
- **پشتیبانی ردیاب UDP**: ارتباط ردیاب UDP سازگار با BEP 15
- **DHT پیشرفته**: جدول مسیریابی کامل Kademlia با جستجوهای تکراری
- **تبادل همتا (PEX)**: کشف همتا سازگار با BEP 11
- **مدیریت اتصال**: انتخاب همتای تطبیقی و محدودیت‌های اتصال
- **بهینه‌سازی‌های پروتکل**: مدیریت پیام کارآمد حافظه با مسیرهای بدون کپی

## افزونه پروتکل Xet (BEP XET)

افزونه پروتکل Xet یک تمایز کلیدی است که BitTorrent را به یک سیستم فایل peer-to-peer فوق‌العاده سریع و قابل به‌روزرسانی بهینه‌شده برای همکاری تبدیل می‌کند. BEP XET امکان می‌دهد:

- **تکه‌بندی تعریف‌شده محتوا**: تقسیم‌بندی هوشمند فایل مبتنی بر Gearhash (تکه‌های 8KB-128KB) برای به‌روزرسانی‌های کارآمد
- **حذف تکراری بین تورنت**: حذف تکراری در سطح تکه در چندین تورنت
- **CAS Peer-to-Peer**: ذخیره‌سازی آدرس‌پذیر محتوا غیرمتمرکز با استفاده از DHT و ردیاب‌ها
- **به‌روزرسانی‌های فوق‌العاده سریع**: فقط تکه‌های تغییر یافته نیاز به توزیع مجدد دارند، امکان اشتراک‌گذاری سریع فایل همکاری
- **سیستم فایل P2P**: تبدیل BitTorrent به یک سیستم فایل peer-to-peer قابل به‌روزرسانی بهینه‌شده برای همکاری
- **تأیید درخت Merkle**: هش‌گذاری BLAKE3-256 با پشتیبان SHA-256 برای یکپارچگی

[بیشتر درباره BEP XET بیاموزید →](bep_xet.md)

### قابلیت مشاهده
- **صادرات متریک**: متریک‌های سازگار با Prometheus برای نظارت
- **ثبت ساختاریافته**: ثبت قابل پیکربندی با ردیابی هر همتا
- **آمار عملکرد**: ردیابی زمان واقعی توان عملیاتی، تأخیر، و عمق صف
- **نظارت بر سلامت**: کیفیت اتصال و امتیازدهی قابلیت اطمینان همتا
- **داشبورد ترمینال**: داشبورد زنده مبتنی بر Textual (Bitonic)
- **مدیر هشدار**: هشدارهای مبتنی بر قانون با پایداری و آزمایش از طریق CLI

UV را از [astral.sh/uv](https://astral.sh/uv) نصب کنید، سپس ccBitTorrent را نصب کنید.

**Bitonic** - رابط داشبورد ترمینال اصلی (توصیه می‌شود):
- راه‌اندازی: `uv run bitonic` یا `uv run ccbt dashboard`

**btbt CLI** - رابط خط فرمان پیشرفته:
- راه‌اندازی: `uv run btbt`

- راه‌اندازی: `uv run ccbt`
