# ccBitTorrent - 고성능 BitTorrent 클라이언트

Python asyncio로 구축된 현대적이고 고성능인 BitTorrent 클라이언트로, 고급 조각 선택 알고리즘, 병렬 메타데이터 교환, 최적화된 디스크 I/O를 특징으로 합니다.

## 기능

### 성능 최적화
- **비동기 I/O**: 우수한 동시성을 위한 완전한 asyncio 구현
- **Rarest-First 선택**: 최적의 스웜 건강을 위한 지능형 조각 선택
- **엔드게임 모드**: 더 빠른 완료를 위한 중복 요청
- **요청 파이프라인**: 깊은 요청 큐 (피어당 16-64개의 미완료 요청)
- **Tit-for-Tat Choking**: 낙관적 언초크를 통한 공정한 대역폭 할당
- **병렬 메타데이터**: 여러 피어로부터 동시 ut_metadata 가져오기
- **디스크 I/O 최적화**: 파일 사전 할당, 배치 쓰기, 링 버퍼 스테이징, 메모리 매핑 I/O
- **해시 검증 풀**: 작업자 스레드 간 병렬 SHA-1 검증

### 고급 설정
- **TOML 설정**: 핫 리로드가 있는 포괄적인 설정 시스템
- **토렌트별 설정**: 개별 토렌트 설정 재정의
- **속도 제한**: 전역 및 토렌트별 업로드/다운로드 제한
- **전략 선택**: 라운드 로빈, rarest-first 또는 순차적 조각 선택
- **스트리밍 모드**: 미디어 파일을 위한 우선순위 기반 조각 선택

### 네트워크 기능
- **UDP 트래커 지원**: BEP 15를 준수하는 UDP 트래커 통신
- **향상된 DHT**: 반복 조회가 있는 완전한 Kademlia 라우팅 테이블
- **피어 교환 (PEX)**: BEP 11을 준수하는 피어 발견
- **연결 관리**: 적응형 피어 선택 및 연결 제한
- **프로토콜 최적화**: 제로 복사 경로를 통한 메모리 효율적인 메시지 처리

## Xet 프로토콜 확장 (BEP XET)

Xet 프로토콜 확장은 BitTorrent를 협업에 최적화된 초고속 업데이트 가능한 피어투피어 파일 시스템으로 변환하는 주요 차별화 요소입니다. BEP XET는 다음을 가능하게 합니다:

- **콘텐츠 정의 청킹**: 효율적인 업데이트를 위한 Gearhash 기반 지능형 파일 분할 (8KB-128KB 청크)
- **토렌트 간 중복 제거**: 여러 토렌트 간 청크 수준 중복 제거
- **피어투피어 CAS**: DHT 및 트래커를 사용한 분산 콘텐츠 주소 지정 가능 스토리지
- **초고속 업데이트**: 변경된 청크만 재배포가 필요하여 빠른 협업 파일 공유 가능
- **P2P 파일 시스템**: BitTorrent를 협업에 최적화된 업데이트 가능한 피어투피어 파일 시스템으로 변환
- **Merkle 트리 검증**: 무결성을 위한 BLAKE3-256 해싱 및 SHA-256 폴백

[BEP XET에 대해 자세히 알아보기 →](bep_xet.md)

### 관찰 가능성
- **메트릭 내보내기**: 모니터링을 위한 Prometheus 호환 메트릭
- **구조화된 로깅**: 피어별 추적이 있는 구성 가능한 로깅
- **성능 통계**: 실시간 처리량, 지연 시간 및 큐 깊이 추적
- **상태 모니터링**: 연결 품질 및 피어 신뢰성 점수
- **터미널 대시보드**: Textual 기반 라이브 대시보드 (Bitonic)
- **경고 관리자**: 지속성 및 CLI를 통한 테스트가 있는 규칙 기반 경고
