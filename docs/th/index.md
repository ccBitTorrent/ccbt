# ccBitTorrent - ไคลเอนต์ BitTorrent ประสิทธิภาพสูง

ไคลเอนต์ BitTorrent ที่ทันสมัยและมีประสิทธิภาพสูงที่สร้างด้วย Python asyncio พร้อมอัลกอริทึมการเลือกชิ้นส่วนขั้นสูง การแลกเปลี่ยนเมตาดาต้าขนาน และ I/O ดิสก์ที่ปรับปรุงแล้ว

## คุณสมบัติ

### การปรับปรุงประสิทธิภาพ
- **Async I/O**: การใช้งาน asyncio แบบเต็มรูปแบบเพื่อการทำงานพร้อมกันที่เหนือกว่า
- **การเลือก Rarest-First**: การเลือกชิ้นส่วนอัจฉริยะเพื่อสุขภาพของฝูงที่เหมาะสมที่สุด
- **โหมด Endgame**: คำขอซ้ำเพื่อการเสร็จสมบูรณ์ที่เร็วขึ้น
- **Pipeline คำขอ**: คิวคำขอที่ลึก (16-64 คำขอที่ค้างอยู่ต่อ peer)
- **Tit-for-Tat Choking**: การจัดสรรแบนด์วิดท์อย่างยุติธรรมด้วย optimistic unchoke
- **เมตาดาต้าขนาน**: การดึง ut_metadata พร้อมกันจากหลาย peer
- **การปรับปรุง Disk I/O**: การจัดสรรไฟล์ล่วงหน้า การเขียนแบบแบทช์ ring-buffer staging, memory-mapped I/O
- **พูลการตรวจสอบ Hash**: การตรวจสอบ SHA-1 แบบขนานใน worker threads

### การกำหนดค่าขั้นสูง
- **การกำหนดค่า TOML**: ระบบการกำหนดค่าที่ครอบคลุมพร้อม hot-reload
- **การตั้งค่าต่อ Torrent**: การแทนที่การกำหนดค่า torrent แต่ละรายการ
- **การจำกัดอัตรา**: ขีดจำกัดการอัปโหลด/ดาวน์โหลดทั่วโลกและต่อ torrent
- **การเลือกกลยุทธ์**: การเลือกชิ้นส่วน round-robin, rarest-first หรือตามลำดับ
- **โหมดสตรีมมิ่ง**: การเลือกชิ้นส่วนตามลำดับความสำคัญสำหรับไฟล์มัลติมีเดีย

### คุณสมบัติเครือข่าย
- **การสนับสนุน UDP Tracker**: การสื่อสาร tracker UDP ที่สอดคล้องกับ BEP 15
- **DHT ที่ปรับปรุงแล้ว**: ตารางการกำหนดเส้นทาง Kademlia แบบเต็มพร้อมการค้นหาแบบวนซ้ำ
- **การแลกเปลี่ยน Peer (PEX)**: การค้นพบ peer ที่สอดคล้องกับ BEP 11
- **การจัดการการเชื่อมต่อ**: การเลือก peer แบบปรับตัวได้และขีดจำกัดการเชื่อมต่อ
- **การปรับปรุงโปรโตคอล**: การจัดการข้อความที่มีประสิทธิภาพหน่วยความจำด้วยเส้นทาง zero-copy

## ส่วนขยายโปรโตคอล Xet (BEP XET)

ส่วนขยายโปรโตคอล Xet เป็นตัวแยกที่สำคัญที่เปลี่ยน BitTorrent เป็นระบบไฟล์ peer-to-peer ที่เร็วมากและอัปเดตได้ซึ่งปรับให้เหมาะสำหรับการทำงานร่วมกัน BEP XET ช่วยให้:

- **การแบ่งส่วนตามเนื้อหา**: การแบ่งส่วนไฟล์อัจฉริยะตาม Gearhash (ชิ้นส่วน 8KB-128KB) สำหรับการอัปเดตที่มีประสิทธิภาพ
- **การลบข้อมูลซ้ำข้าม Torrent**: การลบข้อมูลซ้ำระดับชิ้นส่วนข้ามหลาย torrent
- **CAS Peer-to-Peer**: ที่เก็บข้อมูลที่ระบุที่อยู่เนื้อหาแบบกระจายโดยใช้ DHT และ trackers
- **การอัปเดตที่เร็วมาก**: เฉพาะชิ้นส่วนที่เปลี่ยนแปลงเท่านั้นที่ต้องแจกจ่ายใหม่ ช่วยให้การแชร์ไฟล์การทำงานร่วมกันอย่างรวดเร็ว
- **ระบบไฟล์ P2P**: เปลี่ยน BitTorrent เป็นระบบไฟล์ peer-to-peer ที่อัปเดตได้ซึ่งปรับให้เหมาะสำหรับการทำงานร่วมกัน
- **การตรวจสอบ Merkle Tree**: การแฮช BLAKE3-256 พร้อม fallback SHA-256 สำหรับความสมบูรณ์

[เรียนรู้เพิ่มเติมเกี่ยวกับ BEP XET →](bep_xet.md)

### ความสามารถในการสังเกต
- **การส่งออกเมตริก**: เมตริกที่เข้ากันได้กับ Prometheus สำหรับการตรวจสอบ
- **การบันทึกแบบมีโครงสร้าง**: การบันทึกที่กำหนดค่าได้พร้อมการติดตามต่อ peer
- **สถิติประสิทธิภาพ**: การติดตาม throughput, latency และความลึกของคิวแบบเรียลไทม์
- **การตรวจสอบสุขภาพ**: คุณภาพการเชื่อมต่อและการให้คะแนนความน่าเชื่อถือของ peer
- **แดชบอร์ดเทอร์มินัล**: แดชบอร์ดสดแบบ Textual (Bitonic)
- **ตัวจัดการการแจ้งเตือน**: การแจ้งเตือนตามกฎพร้อมความคงทนและการทดสอบผ่าน CLI
